# DTensor-Supported APIs

<!-- md-trans-meta sourceCommit=unknown translatedAt=2026-06-15T02:24:22.667Z pushedAt=2026-06-15T03:25:49.241Z -->

For the list of DTensor-supported APIs, see [Table 1](#dtensor-supported-api-list).

> [!NOTE]  
>
> - **DTensor** (Distributed Tensor) is a distributed tensor abstraction provided by PyTorch, and is a subclass of `torch.Tensor`. DTensor follows the Single Program, Multiple Data (SPMD) programming model, using `DeviceMesh` and `Placement` (including `Shard`, `Replicate`, `Partial`) to describe the sharding layout of tensors across multiple devices. It transparently handles sharded storage, operator computation, and cross-device collective communication, enabling users to write distributed programs as if writing single-device programs. For details, see the [PyTorch DTensor official documentation](https://pytorch.org/docs/stable/distributed.tensor.html).
> - The list presented in this document includes APIs that have been verified to support DTensor. Some unverified APIs may also have DTensor support capability. However, use them with caution in the Ascend NPU environment. We will continue to verify more APIs and update this document regularly.
